Brief overview of game engines

Using a game engine is different from creating ordinary programs. Instead of writing all of our code from scratch, we extend the basic functionality the engine provides.

Godot comes with two main things:

  1. Rendering - drawing stuff on the screen
  1. Physics - making stuff collide with other stuff

Both of those things are super hard to do from scratch, which is why we use a game engine 😛

Nodes

Godot is built off of “nodes”.

Nodes can have children — other nodes are “attached” to the parent. Usually this means their positions are linked or their functionality is linked. Examples:

  1. A game character holding an object
  1. A cloud with extra logic for rain and lightning

Extending functionality

Godot provides us with a RigidBody2D node, which acts as a physics body (impacted by gravity, momentum, etc.)

However, we want extra control over how our RigidBody2D moves, so we can attach a script to the node.

Our script looks something like this: (sorry for no code highlighting 😭)

extends RigidBody2D

var upward_force = Vector2(0.0, 1.0)

func _process(delta: float) {
	apply_central_force(upward_force)
}

Let’s break this down:

  1. extends RigidBody2D This line of code tells Godot that the following script is attached to a RigidBody2D node. It lets us use the existing functions of this node, like apply_central_force
  1. var upward_force = Vector2(0.0, 1.0) Here, we define a variable. “Vector2” is a built-in class used to represent positions in 2D space. A Vector2 has an x-component and a y-component.
  1. func _process(delta: float) GDScript functions are defined with the “func” keyword. ”_process” is a special function that Godot will execute every frame. This is great for behavior that happens all the time.
  1. apply_central_force(upward_force) Now we call an existing function of RigidBody2D. The function takes one parameter, which is the direction of the force, to which we input the variable we created in step 2.
